# Post-Event Review Checklist

**Event type:** [OADC trigger / duress event / SE attempt / break-glass / H-3 or H-4 / team degradation]
**Event date:** YYYY-MM-DD
**Review lead:** [Name]
**Review due by:** [Date per §E.2 timelines]

## Review items

- [ ] Event timeline reconstructed from decision logs and evidence
- [ ] OADC condition that triggered identified and documented
- [ ] Threshold accuracy assessed: did the trigger activate at the right time?
- [ ] Authority response assessed: was the response proportionate and effective?
- [ ] Handoff (if applicable): was context transfer complete?
- [ ] System enforcement (if applicable): did technical controls activate correctly?
- [ ] Operator welfare addressed (if H-3/H-4 or duress)
- [ ] Parameter adjustment recommendations documented (if any)
- [ ] Corrective actions assigned with owners and deadlines
- [ ] Pattern check: does this repeat a pattern from previous reviews?
- [ ] Review record stored in evidence pipeline
